# V2 Deployment Readiness Checklist

## Deployment scope
- Environment: testnet or prod
- Profile: conservative / normal / aggressive
- Mode: shadow / live
- Report generated by `v2/scripts/preflight.sh`

## Before deployment
- [ ] `.env` contains only secrets (`BINANCE_API_KEY`, `BINANCE_API_SECRET`, webhooks) and no behavior flags.
- [ ] Secrets are not tracked by git history and no sample key values are committed.
- [ ] `v2/config/config.yaml` profile and inheritance (`conservative`, `normal`, `aggressive`) are reviewed before changes.
- [ ] `python -m ruff check v2 tests/v2` passes.
- [ ] `python -m pytest -q tests/v2` passes.
- [ ] `v2/scripts/preflight.sh` run completed and report is PASS.
- [ ] `--keep-reports` retention policy is set and applied to report directory as needed.

## Runtime checks (shadow first)
- [ ] Shadow mode has been run for at least `N` days in CI/staging simulation.
- [ ] Rebuild/start script in `README` and this runbook executed successfully.
- [ ] Pause flow tested: `--ops-action pause` returns `{ "action": "pause", "paused": true }`.
- [ ] Safe mode flow tested: `--ops-action safe_mode` returns `paused=true`, `safe_mode=true`.
- [ ] Resume flow tested: `--ops-action resume` returns `paused=false`, `safe_mode=false`.
- [ ] Flatten flow tested at least once: return has `open_regular_orders=0`, `open_algo_orders=0`, `position_amt=0.0`.

## Order lifecycle and risk checks
- [ ] TP/SL placement is verified through Algo service on testnet.
- [ ] No legacy conditional order endpoint usage detected for TP/SL (`/fapi/v1/order` with STOP/MARKET condition).
- [ ] Time sync drift check passed in preflight (`/fapi/v1/time` vs local time).

## Binance error readiness
- [ ] No unresolved `-4120` path: conditional order logic is routed through algo endpoint.
- [ ] No unresolved `-4015` risk: `clientAlgoId/clientOrderId` format and length policy confirmed.
- [ ] No unresolved `-1021` risk: time sync and retry strategy confirmed.
- [ ] Rate-limits monitored: no repeated `429`/`418` storms after tuning.

## Final sign-off
- [ ] Preflight report path recorded:
  - `v2/reports/deployment_readiness_YYYYMMDD_HHMMSS.md`
- [ ] Old reports retention policy applied (예: `--keep-reports 30`) if report archive has growth policy.
- [ ] Deployment Lead completed final checklist and signed off by date.
- [ ] Rollback plan prepared: stop process, switch safe mode / flatten if needed, restore env.
