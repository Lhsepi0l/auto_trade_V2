# V2 초간단 쌈뽕 사용법 (진짜 쉬움)

## 0. 이 문서 한 줄 요약
- **버튼만 순서대로 누르면 자동매매가 돌아간다.**

## 1. 시작 전에 딱 3가지만 확인
1. `v2-stack.service`가 켜져 있다.
2. 웹 운영 패널(`/operator`)이 보인다.
3. 잔고가 보인다. (예: 35 USDT)

끝. 이 3개면 시작 가능.

## 2. 버튼 누르는 순서 (그대로 따라하기)

### STEP 1) `고급설정` 누르기
- 고급 화면으로 들어간다.

### STEP 2) `증거금 설정` 누르기
- 값 입력: `10` (처음 테스트는 작게)

### STEP 3) `심볼 설정` 누르기
- 값 입력: `BTCUSDT,ETHUSDT`

### STEP 4) `리스크 기본` 누르기
- 아래 값 입력:
  - `최대 레버리지 (max_leverage)` = `5`
  - `최대 노출 비율 (max_exposure_pct)` = `35`
  - `최대 포지션 배수 (max_notional_pct)` = `1000`
  - `1회 트레이드 리스크 % (per_trade_risk_pct)` = `1.0`

### STEP 5) `심볼 레버리지` 누르기
- 2번 입력:
  - `BTCUSDT` / `5`
  - `ETHUSDT` / `4`

### STEP 6) `즉시 판단` 버튼 2~3번
- `no_candidate`가 나와도 정상이다. (조건 안 맞아서 대기 중)

### STEP 7) `시작` 버튼
- 이제 자동으로 기회 기다리다가 조건 맞으면 진입한다.

## 3. 지금 봐야 할 정상 신호
- `즉시 판단: no_candidate` -> 정상 (대기)
- `실시간 잔고: 사용가능 ...` -> 정상
- `(최근 캐시)` -> 일시 네트워크 흔들림일 때 정상 보정 표시

## 4. 진입 후 자동으로 되는 것
- TP/SL 브래킷 주문 2개 자동 생성
- 한쪽 체결되면 반대쪽 자동 취소
- 포지션 0 되면 남은 주문 자동 정리

즉, **진입 후 익절/손절은 자동**이다.

## 5. 절대 하지 말 것 (중요)
- 화나서 설정을 계속 바꾸지 말기
- 시작하자마자 레버리지 크게 올리지 말기
- 잔고 적을 때 심볼 너무 많이 늘리지 말기

## 6. 문제 생기면 이렇게만 보내기
아래 3줄 복붙해서 보내면 바로 점검 가능:

```text
즉시 판단: ...
결과: ...
실시간 잔고: ...
```
