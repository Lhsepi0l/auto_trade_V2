# 전략 현실 점검

## 1. 결론
2026-03-28 현재 기준으로 이 전략은
- **운영 안정성은 많이 좋아졌지만**
- **수익성이 증명된 돈 버는 시스템이라고 말할 단계는 아니다.**

즉,
- "실거래에서 바로 죽는 구조"는 줄어들었고
- "지금 당장 월 3000달러급 우상향"이라고 자신 있게 말할 근거는 아직 부족하다.

## 2. 지금까지 실제로 좋아진 것

### 2.1 운영 안정성
- TP/SL 브래킷이 진입 후 사라지던 버그를 막았다.
- 레버리지 입력이 실제 주문 leverage에 맞게 반영되도록 고쳤다.
- 기존 포지션 보유 중 stale 출렁임 원인을 줄였다.
- ntfy 알림이 실패처럼 과장되게 보이던 부분을 정리했다.

즉, 시스템은 예전보다 훨씬 덜 불안정해졌다.

### 2.2 전략 구조
- `alpha_expansion` 쪽에서 “무조건 더 많이 들어가자”가 아니라
  “애매한 확장보다 좋은 확장을 더 잘 살리자” 방향으로 리팩터링했다.
- 현재 live candidate의 핵심 구조는 다시
  - `15m` 실행
  - `1h` bias
  - `4h` regime
  3-TF core로 정리했다.

## 3. 2026-03-28 전략 검토에서 확인한 사실

### 3.1 30m / 2h 직접 보강 버전
- 방향성 실험으로는 의미가 있었다.
- 하지만 fixed-window 1Y 비교에서는
  - 거래 수는 늘었지만
  - `net`, `PF`, `DD`가 baseline보다 나빠졌다.

즉:
- “더 많이 들어가는 것”은 만들었지만
- “더 똑똑하게 들어가는 것”으로는 이어지지 않았다.

그래서 이 버전은 **live 기본 경로에 채택하지 않았다.**

### 3.2 현재 채택한 현실적 개선안
- `15m/1h/4h` 유지
- `expansion_quality_score_v2_min=0.62` 적용
- `edge_ratio >= 0.95 + high quality + 정상 spread/stop`인 cost near-pass 허용

의미:
- 좋은 자리와 애매한 자리를 조금 더 구분하려는 보수적 개선
- 구조를 과하게 흔들지 않는 선에서 품질을 더 보겠다는 선택
- 단순히 거래를 줄이는 게 아니라, fixed-window 1Y에서 실제로
  - `net=6.14 -> 6.73`
  - `PF=2.707 -> 3.127`
  - `DD=5.62% -> 4.48%`
  - `trades=318 -> 281`
  로 개선된 sweet spot 값이다.
- 추가 6개월 검증에서도
  - `2025-03-28 ~ 2025-09-27`은 소폭 개선
  - `2025-09-28 ~ 2026-03-28`은 사실상 동일
  이라서 “해를 만들지 않으면서 애매한 expansion을 더 자르는 값”으로 볼 수 있다.
- 여기에 `cost near-pass`를 더했을 때도
  - 1Y / 6M 성과는 그대로 유지됐고
  - `cost_missing` 건수만 줄어들었다.
  즉, 현재는 `trigger`보다 `cost edge_shortfall` 쪽이 더 실제 병목에 가깝다고 볼 수 있다.

## 4. 왜 아직 “돈 버는 기계”라고 말하면 안 되나

### 4.1 검증 범위 부족
- 지금은 고정 구간 local backtest 기준으로 비교했다.
- 이것만으로는
  - 다른 구간
  - 다른 시장 상태
  - 실거래 슬리피지/심리/운영 변수
까지 다 증명되진 않는다.

### 4.2 전략 개선의 성격
- 지금까지 한 일은 mostly
  - 운영 안정화
  - 잘못된 진입/오류 경로 제거
  - 애매한 신호 구조 정리
에 가깝다.

이건 중요하지만,
곧바로 “고수익 보장”으로 이어지는 건 아니다.

## 5. 지금 전략을 어떻게 봐야 하나

### 맞는 표현
- 운영 가능한 실험 시스템
- 안정성은 꽤 좋아진 상태
- 아직 수익성 최적화는 진행 중인 상태

### 틀린 표현
- 월 3000달러가 검증된 시스템
- 계속 우상향이 보장된 전략
- 이미 완성된 실거래 최종판

## 6. 다음에 집중해야 할 것
이제 중요한 건 지표를 더 붙이는 게 아니라,

1. 좋은 expansion 자리 정의를 더 날카롭게 하기
2. 애매한 trigger를 더 잘 자르기
3. fixed-window 1Y 뿐 아니라 추가 구간에서도 일관성 확인하기

2026-03-29 기준으로는 위 3개 중
- `1`은 `qv2_min=0.62` 채택으로 1차 반영했고
- `3`은 1Y + 2개 6개월 창으로 추가 확인했다.

다음 집중 포인트는 이제
- 단순 문턱값 조정보다
- `trigger_missing`과 `volume_missing`이 많이 나오는 애매한 구간을 더 구조적으로 줄이는 쪽이다.

추가 메모:
- 2026-03-29에 narrow trigger/volume rescue 구조도 실험했지만,
  1Y와 6M fixed-window에서 모두 수치가 소폭 후퇴해 live 기본 경로에는 채택하지 않았다.
- 즉 다음 단계는 rescue heuristics를 더 붙이는 게 아니라,
  실제 near-miss 샘플을 뽑아 `buffer / cost / width` 중 무엇이 진짜 병목인지 다시 설계하는 쪽이 맞다.
- 이를 위해 `python -m v2.backtest.block_samples ...` 표본 추출기를 추가했고,
  첫 1Y 리포트에서 `trigger_missing`의 near-break + body/close/width 정합 샘플이 `long=261`, `short=255` 수준으로 확인됐다.
  반면 `volume_missing`는 평균 `vol_ratio≈0.551`이라 대다수는 진짜 저볼륨으로 보는 게 맞다.
- 이 표본에서 가장 유망해 보였던 규칙
  - `buffer 8bps 안쪽`
  - `body/close/width/squeeze 정합`
  만 별도로 임시 적용해서 다시 돌려봤지만,
  1Y/6M 모두 기대수익이 아주 소폭 후퇴해 채택하지 않았다.
- 반대로 `cost_missing`는 `354`건 중 `353`건이 `edge_shortfall`이었고,
  `edge_ratio>=0.95` near-pass 표본이 의미 있게 존재했다.
  이 규칙은 실제로 넣어도 성과를 해치지 않아서 채택했다.
- 추가로 near-buffer 후보 `203`개를 setup bucket으로 자동 분류한 결과, 현재 top 3는 아래처럼 정리됐다.
  - `broad_quality`: `gap<=8bps`, `body>=0.35`, `favored>=0.60`, `width>=0.05`, `range_atr>=0.85`, `count=106`
  - `balanced_range`: `gap<=5bps`, `body>=0.40`, `favored>=0.65`, `width>=0.05`, `range_atr>=0.90`, `count=57`
  - `balanced_core`: `gap<=3.5bps`, `body>=0.45`, `favored>=0.68`, `width>=0.08`, `range_atr>=0.95`, `count=27`
- 다음 `setup/confirm` 실험 1순위는 support/quality 균형상 `balanced_range`로 잡는 것이 맞다.

한 줄 요약:
- **시스템은 좋아졌다**
- **전략은 아직 더 좁혀야 한다**
- **과장 대신 검증으로 가야 한다**
